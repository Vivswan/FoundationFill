Foundation Fill

一個用於填充文本字段的 Chrome 擴展，提供預定義模板和使用 LLM API 生成文本的功能

概述

Foundation Fill 幫助您快速使用自定義模板填充文本字段。它特別適用於：

  • 軟件工程師創建樣板代碼片段
  • 作家為 AI 工具使用一致的提示
  • 支持團隊使用標準化回覆模板
  • 研究人員為 AI 助手使用特定指令

功能

模板管理

創建、編輯、複製、重新排序和管理可重用模板，方便快速訪問。

特定域名模板

配置模板僅在特定網站上顯示，以獲得更好的組織管理。

模板變量

在模板中使用 {{變量:預設值}} 語法，實現自定義值。

右鍵菜單集成

在任何文本字段上右鍵單擊即可快速訪問您的模板。

主題支持

選擇淺色/深色/系統主題，並提供多種強調色選項。

多語言支持

使用英語、簡體中文或繁體中文界面。

API 生成

通過自定義 API 集成生成文本，生成過程中顯示動畫效果。

開始使用

安裝

從 Chrome 網上應用店安裝（推薦）

  1. 訪問Chrome 網上應用店頁面
  2. 點擊"添加至 Chrome"
  3. 接受權限

從 GitHub 發布頁安裝

  1. 訪問GitHub 發布頁
  2. 下載最新的發布 zip 文件
  3. 將 zip 文件解壓到一個文件夾
  4. 前往 chrome://extensions/
  5. 啟用開發者模式
  6. 點擊"載入已解壓的擴展程序"並選擇解壓後的文件夾

手動安裝（開發）

  1. 克隆倉庫或下載源代碼
  2. 運行 npm install 安裝依賴
  3. 運行 npm run build 構建擴展
  4. 前往 chrome://extensions/
  5. 啟用開發者模式
  6. 點擊"載入已解壓的擴展程序"並選擇 dist 目錄

使用指南

創建模板

  1. 點擊瀏覽器工具欄中的 Foundation Fill 圖標
  2. 點擊"新建"按鈕創建新模板
  3. 輸入您的系統提示和用戶提示
  4. 您的模板將自動保存

使用模板

  1. 在瀏覽網頁時右鍵點擊任何文本字段
  2. 從"Foundation Fill"下的右鍵菜單中選擇您的模板
  3. 模板內容將被插入到文本字段中

組織模板

  1. 在擴展彈出窗口中，使用手柄圖標拖放模板
  2. 預設模板將始終保持在頂部
  3. 使用複製按鈕（複製圖標）創建任何模板的副本
  4. 複製的模板將直接放置在原始模板之後

特定域名模板

  1. 編輯模板並點擊"域名"按鈕
  2. 點擊"添加當前域名"或手動輸入域名
  3. 模板將只在指定的網站上顯示
提示： 使用域名通配符（例如，*.example.com）匹配所有子域名。

使用模板變量

  1. 在模板中使用 {{變量:預設值}} 格式添加變量
  2. 使用模板時，系統會提示您提供值
  3. 具有相同名稱的變量將在整個模板中使用相同的值

使用 API 生成文本

  1. 點擊齒輪圖標進入設置
  2. 輸入您的 API 配置（URL、密鑰、模型）
  3. 使用"測試提示"按鈕通過模板生成文本

設置

API 配置

配置您首選的 LLM API 用於文本生成：

  • 基礎 URL： API 端點（例如，https://api.openai.com/v1）
  • API 密鑰： 您的認證密鑰（安全存儲）
  • 模型： 模型標識符（例如，gpt-4）

主題設置

自定義擴展的外觀：

  • 主題： 淺色、深色或系統（跟隨瀏覽器的主題）
  • 強調色： 選擇藍色、紅色、綠色、紫色、橙色或粉色
  • 語言： 選擇英語、簡體中文或繁體中文

導入/導出

備份和恢復您的模板和設置：

  • 導出所有數據： 將您的模板和設置保存為 JSON 文件
  • 導入數據： 從之前導出的文件恢復

架構

Foundation Fill 遵循具有面向服務組件的模型-視圖架構：

核心組件

  • 模型： 處理數據和業務邏輯（模板、設置）
  • 視圖： 渲染用戶界面並處理用戶交互
  • 服務： 提供可重用功能（存儲、API、主題）
  • 後台腳本： 管理右鍵菜單和消息傳遞
  • 內容腳本： 填充文本字段並管理文本插入

數據流

模板存儲在 Chrome 存儲中，由後台腳本訪問以填充右鍵菜單。當選擇模板時，如果需要，會處理變量，然後發送到內容腳本以插入到活動文本字段中。
